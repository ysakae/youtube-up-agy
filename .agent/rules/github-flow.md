# Agent Rules for this Project

このプロジェクトでは GitHub Flow に従った開発プロセスを遵守してください。

## 開発プロセス (GitHub Flow)

1.  **ブランチ作成 (Create a Branch)**
    *   作業を開始する際は、必ず `main` ブランチから最新を取得し、新しい機能ブランチ（feat/ または fix/）を作成すしてください。
    *   ブランチ名は作業内容を表す記述的な名前にしてください（例: `feature/add-login`, `fix/header-style`）。
    *   `main` ブランチへ直接コミットすることは避けてください。

2.  **コミット (Add Commits)**
    *   変更は論理的な単位で小分けにコミットしてください。
    *   コミットメッセージは変更内容がわかるように明確に記述してください。

3.  **検証 (Verification)**
    *   **新規機能の追加や既存機能の改修を行う際は、必ずそれに伴うユニットテストも合わせて実装・修正すること（テストコードの実装を必須とします）。**
    *   コード変更後は、内蔵ブラウザやターミナルを使用してテストを実行し、その結果を Artifacts（スクリーンショット等）として報告してください。

4.  **プルリクエスト (Open a Pull Request)**
    *   機能実装や修正が完了したら（あるいは途中でもフィードバックが欲しい場合）、プルリクエストを作成することを想定して作業を進めてください。
    *   コードレビューを経て、承認された変更のみを `main` にマージします。
    *   プルリクエストの作成には gh CLI (GitHub CLI) を使用してください。

5.  **デプロイとマージ (Discuss and Review, Deploy, Merge)**
    *   マージ後、不要になったブランチは削除します。
